# PDF 列印與排版重新設計計畫書

此計畫書針對使用者反應的「PDF 功能空間浪費、排版奇怪」問題，提出專屬印表樣式（Print Stylesheet）優化方案。我們將徹底重構 `@media print` 區塊，以實現高密度、頁面對齊、無多餘留白的 A4 紙張列印效果。

## 使用者審查與討論

請確認以下 PDF 列印與排版重新設計細節：

> [!IMPORTANT]
> **排版異常與空間浪費的主要原因分析**：
> 1. **預設響應式斷點失效**：印表機輸出時，瀏覽器會以虛擬寬度（通常介於 800px ~ 1024px）渲染。這導致原本在螢幕上是 3 欄的卡片網格，在列印時被判定為中窄螢幕，自動縮成**單欄排列**，造成左右兩側有巨大的空白，且整份文件被拉長至 4~5 頁。
> 2. **卡片跨頁被截斷**：沒有防止卡片內折頁，導致某張卡片的標題在第一頁底部，而內容卻被切到第二頁頂部。
> 3. **瀏覽器邊距過大**：未指定 `@page` 邊距，導致瀏覽器套用預設的寬邊距，浪費了將近 30% 的可列印面積。

## Proposed Changes (列印排版重構方案)

我們將在 [index.html](file:///c:/Users/admin/Documents/Workspace/Machine-Learning-Study/infographic/index.html) 中加入精細的 `@media print` 控制：

### 1. 定義 A4 紙張與邊距範圍
* 設定標準 `@page` 邊距為 `10mm`，最大化可列印面積。
* 強制移除瀏覽器預設頁首頁尾產生的 URL 與日期時間（透過設定 `@page { margin: 10mm; }` 配合隱藏元素）。

### 2. 強制列印雙欄排版（卡片檢視）
* 在列印時，無視螢幕寬度，強制 `#grid-view-container` 採用 `grid-template-columns: repeat(2, 1fr) !important`（雙欄並排）。
* 10 張卡片剛好會呈現 5 列（第一頁 3 列共 6 張卡片，第二頁 2 列共 4 張卡片，加上底部的 Roadmap 與 Pipeline）。這樣排版將非常緊湊且高密度。

### 3. 防止卡片與區塊跨頁折斷
* 對 `.glass-card` 套用 `break-inside: avoid-page !important` 與 `page-break-inside: avoid !important`，確保任一張模型卡片不會被攔腰截斷。
* 對底部的「專業技能路徑 (Roadmap)」與「商業價值管道 (Pipeline)」套用防折斷，確保它們作為一個完整區塊出現在第二頁底部。

### 4. 調校列印字型大小與內距 (Density Optimization)
* 在列印時將基礎文字縮小 10%~15%（卡片內距從 `p-5` 縮小為 `p-3.5`，文字從 `text-sm` 降為 `text-xs`，說明字體微調為 `text-[10px]`）。
* 精簡卡片間的間距，將 `gap-6` 縮小為 `gap-3.5`，確保版面完美塞滿兩頁 A4，不留任何多餘頁面。

### 5. 視覺包裝 (UI/UX Premium Aesthetics)
* **預設為亮色紙張模式 (Default Light Mode)**：將網頁載入時的預設主題從暗色系改為亮色系（`light-mode`），以貼近實體書本/報告的簡潔紙張感；使用者仍可透過頂部按鈕切換為暗色螢幕模式。
* **色彩系統**：使用深沉且富有科技感的暗色背景（#090d16），搭配高發光的青色（Cyan）、傳統紙張色與灰色邊框等。
* **玻璃擬態卡片**：使用 `backdrop-filter: blur` 搭配極細邊框與陰影，呈現極致的現代感。
* **PDF 一致性印表樣式**：確保網頁的漸變背景、發光框線在列印 PDF 時依然完美保留。

### 6. 表格檢視 (Table View) 列印最佳化
* 當使用者切換至「詳細表格」列印時，表格寬度強制 `100%`，並壓縮欄位內距與字體大小，確保不會超出 A4 紙張右側邊界，且不會產生水平滾動截斷。
